设备运维学习平台 v5 - 免费部署方案

当前状态：Netlify 部署额度已用尽，请选择以下任一免费方案重新部署。

方案一：Vercel（推荐）

优点：免费、自动 HTTPS、全球 CDN、支持 Git 自动部署

步骤：

  1. 访问 vercel.com，用 GitHub/GitLab 账号登录
  2. 点击 "New Project" → "Import Git Repository"
  3. 创建新仓库并上传文件（或直接使用下方提供的 GitHub 仓库）
  4. Vercel 自动检测为静态网站，点击 Deploy
  5. 获得类似 https://your-project.vercel.app 的地址

创建 GitHub 仓库 前往 Vercel

方案二：Cloudflare Pages

优点：免费、无限带宽、全球 CDN、支持自定义域名

步骤：

  1. 访问 pages.cloudflare.com 注册/登录
  2. 连接 GitHub/GitLab 账号
  3. 选择仓库或创建新仓库上传文件
  4. 构建命令留空（纯静态 HTML）
  5. 获得 https://your-project.pages.dev 地址

前往 Cloudflare Pages

方案三：GitHub Pages（最简单）

优点：完全免费、无需第三方服务、GitHub 原生支持

步骤：

  1. 在 GitHub 创建新仓库（如 device-maintenance）
  2. 上传 device_platform_v5.html 并重命名为 index.html
  3. 进入仓库 Settings → Pages → Build and deployment
  4. Source 选择 "Deploy from a branch" → Branch: main，文件夹: / (root)
  5. 等待几分钟，访问 https://[用户名].github.io/[仓库名]
# 快速命令（需安装 Git）
git init
git add device_platform_v5.html
git mv device_platform_v5.html index.html
git commit -m "Add device platform"
git branch -M main
git remote add origin https://github.com/[用户名]/[仓库名].git
git push -u origin main

创建 GitHub 仓库

方案五：本地局域网共享（无公网）

适用场景：团队在同一局域网内使用

步骤：

  1. 在一台电脑上运行简易 HTTP 服务器：
    2. # Python 3
    python -m http.server 8000

    # Node.js (需安装 http-server)
    npx http-server -p 8000

    # 或使用任何现成工具（如 HFS, XAMPP）
  2. 获取本机 IP 地址（cmd 输入 ipconfig，找 IPv4 地址）
  3. 其他人浏览器访问 http://[IP地址]:8000
  4. MQTT 协同功能仍可正常使用（通过公共 broker）

功能验证

部署后请按以下步骤验证：

  1. 用两个不同设备/浏览器打开部署地址
  2. 双方都显示"协同：已连接"（MQTT 连接成功）
  3. 一方上传 PDF/Word 文件到"故障手册"
  4. 另一方等待几秒后点击文件，应在新窗口正常打开
  5. 测试工作日志添加、删除、同步功能

如遇问题，检查浏览器控制台（F12）是否有错误。

推荐选择：方案三（GitHub Pages）最简单可靠，方案一（Vercel）性能最好。
